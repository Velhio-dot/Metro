# HANDOVER: GPU 2D LIGHTING SYSTEM (UNITY 6)

## Обзор системы
Реализована кастомная система 2D освещения на базе GPU Raymarching, полностью совместимая с **Unity 6 Render Graph API**.

### Ключевые компоненты:
1. **GPU2DLightFeature.cs**:
   - Использует `RTHandle` для персистентности (текстура не мерцает и не пропадает).
   - Захватывает объекты на слое `occlusion`.
   - Метод захвата: `RendererList` с поддержкой тегов `Universal2D`, `Always`, `SRPDefaultUnlit` и `UniversalForward`.
2. **GPU2D_Lit_Pro.shader**:
   - Реализует **2.5D освещение**. 
   - `_IsWall`: Если включено, высота пикселя вычисляется как `uv.y * _WallHeight`.
   - `Raymarching`: Луч света летит от `pixelHeight` до `_LightHeight` (высота фонарика игрока).
   - `Dithering`: Добавлен шум `InterleavedGradientNoise` для сглаживания ступенчатых теней.
   - `Shadow Bias`: Используется для предотвращения самозатенения стен.
3. **GPU2DLightManager.cs**:
   - Глобальный передатчик позиции света в шейдеры (`_LightPos`, `_LightRadius` и т.д.).
4. **GPU2D_Diagnostic.cs**:
   - Инструмент для проверки, видит ли система объекты на нужном слое.

## Инструкция по настройке «Г-образных» теней (задача на завтра):
Для динамических теней колонн:
- Материал **Пола**: `Is Wall = OFF`, `Shadow Bias = 0.02`.
- Материал **Колонны/Стены**: `Is Wall = ON`, `Shadow Bias = 0.05` (подбирать до исчезновения черных пятен на лице стены).
- **Z-Heights**: Оптимально `Light Z = 0.5`, `Wall Z = 1.0`.

## Текущее состояние:
Система полностью стабильна в Unity 6. Ошибки захвата устранены. Идет этап визуальной настройки объема (L-shaped shadows).
